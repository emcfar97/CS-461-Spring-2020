
This was definitely a pretty frustrating program, with roadblock after roadblock halting all progress. 
Though I quickly got the hang of clojure, I struggled mightily through several snags until I finally had to give up because I could not, for the life of me, figure out how to install the claypoole dependency, which is required for specifying the number of threads in a pool. I wanted to do the project in clojure and get that experience with a unfamilir language, but I was just wasting my time tring to pull blood from this stone. 
I had much more success with python, recreating what before took days in mere hours. But even then, I ran into several novel problems, all of which were navigable, save one. Apprently, python's multiprocessing module does not like lambda functions. As a result, I had to mangle the code for my entropy function by adding list comprehensions between the mapping functions. When it comes to functional programming, I don't know how kosher this is, but at this point I  just desperate for this whole thing to be over.
Looking at the result graph, it seems adding more concurrent threads improves speed up most significantly for the character triples, which is unsurprising. For all character spans, the sweet spot seems to be between 4 and 8 threads for most improvement. After that point, more threads start to hurt performance, though never as badly as no threads.
